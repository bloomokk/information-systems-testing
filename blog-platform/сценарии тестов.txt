Сценарий тестов

Тест 1. Проверка создания поста

Название теста: Проверка создания поста через API

Цель тестирования: Убедиться, что по API можно создать новый пост и что созданный пост возвращается с корректными полями и доступен по GET-запросу.

Предпосылки:
  Сервер приложения запущен.
  API доступен по базовому URL (например, http://127.0.0.1:8000).
  База данных доступна (тестовая или основная).

Шаги тестирования:

1. Отправить POST-запрос на /api/posts с телом JSON: title «Первый пост», content «Текст поста», author «Иван».

   Ожидаемый результат: Сервер возвращает код ответа 201 (Created). В теле ответа (JSON) присутствуют поля: id, title «Первый пост», content «Текст поста», author «Иван», created_at.

2. Отправить GET-запрос на /api/posts/{id}, где id — значение из ответа шага 1.

   Ожидаемый результат: Сервер возвращает код 200 (OK). В ответе заголовок поста равен «Первый пост».

Фактический результат: Соответствует ожидаемому. test_create_post PASSED.

Тест 2. Проверка редактирования поста

Название теста: Проверка редактирования поста через API

Цель тестирования: Убедиться, что существующий пост можно изменить через PUT-запрос и что в ответе возвращаются обновлённые данные.

Предпосылки:
  Сервер приложения запущен.
  API доступен.
  База данных доступна.

Шаги тестирования:

1. Создать пост: отправить POST /api/posts с телом: title «Пост», content «Текст», author «Автор».

   Ожидаемый результат: Код ответа 201. В ответе есть поле id поста.

2. Отправить PUT-запрос на /api/posts/{id} (id из шага 1) с телом: title «Обновлённый пост», content «Новый текст», author «Автор».

   Ожидаемый результат: Сервер возвращает код 200 (OK). В JSON-ответе поле title равно «Обновлённый пост», поле content равно «Новый текст».

Фактический результат: Соответствует ожидаемому. test_update_post PASSED.

Тест 3. Проверка удаления поста

Название теста: Проверка удаления поста через API

Цель тестирования: Убедиться, что пост можно удалить через DELETE-запрос и что после удаления запрос GET по этому id возвращает 404.

Предпосылки:
  Сервер приложения запущен.
  API доступен.
  База данных доступна.

Шаги тестирования:

1. Создать пост: отправить POST /api/posts с телом: title «Удаляемый пост», content «Текст», author «Автор».

   Ожидаемый результат: Код ответа 201. В ответе есть id поста.

2. Отправить DELETE-запрос на /api/posts/{id} (id из шага 1).

   Ожидаемый результат: Сервер возвращает код 204 (No Content).

3. Отправить GET-запрос на /api/posts/{id} для того же id.

   Ожидаемый результат: Сервер возвращает код 404 (Not Found). Пост не найден.

Фактический результат: Соответствует ожидаемому. test_delete_post PASSED.

  Сервер приложения запущен.
  API доступен.
  База данных доступна.

Шаги тестирования:

2. Отправить POST-запрос на /api/posts/{post_id}/comments с телом: author «Комментатор», content «Мой комментарий».

   Ожидаемый результат: Код 200. В списке один комментарий. У первого комментария content равен «Мой комментарий».

Фактический результат: Соответствует ожидаемому. test_create_comment PASSED.
