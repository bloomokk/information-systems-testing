Сценарий тестов

1. test_create_post — проверка создания поста.

действия:
  1. Отправка POST-запроса на /api/posts с телом: заголовок «Первый пост», текст «Текст поста», автор «Иван».
  2. Проверка кода ответа и полей в JSON-ответе.
  3. Отправка GET-запроса на /api/posts/{id} созданного поста.

ожидаемый результат:
  1. Сервер возвращает код ответа 201 (Created).
  2. В ответе присутствуют поля: id, title «Первый пост», content «Текст поста», author «Иван», created_at.
  3. GET-запрос возвращает код 200 и тот же заголовок «Первый пост».


2. test_update_post — проверка редактирования поста.

действия:
  1. Создание поста через POST /api/posts (заголовок «Пост», текст «Текст», автор «Автор»).
  2. Отправка PUT-запроса на /api/posts/{id} с новыми данными: заголовок «Обновлённый пост», текст «Новый текст», автор «Автор».
  3. Проверка ответа.

ожидаемый результат:
  1. POST возвращает 201, в ответе есть id поста.
  2. PUT возвращает код 200 (OK).
  3. В JSON-ответе PUT присутствуют заголовок «Обновлённый пост» и текст «Новый текст».


3. test_delete_post — проверка удаления поста.

действия:
  1. Создание поста через POST /api/posts (заголовок «Удаляемый пост», текст «Текст», автор «Автор»).
  2. Отправка DELETE-запроса на /api/posts/{id}.
  3. Отправка GET-запроса на /api/posts/{id} того же поста.

ожидаемый результат:
  1. POST возвращает 201.
  2. DELETE возвращает код 204 (No Content).
  3. GET по удалённому посту возвращает код 404 (Not Found).


4. test_create_comment — проверка создания комментария к посту.

действия:
  1. Создание поста через POST /api/posts (заголовок «Пост», текст «Текст», автор «Автор»).
  2. Отправка POST-запроса на /api/posts/{post_id}/comments с телом: автор «Комментатор», текст «Мой комментарий».
  3. Проверка ответа и отправка GET /api/posts/{post_id}/comments.

ожидаемый результат:
  1. Создание поста возвращает 201, в ответе есть id поста.
  2. POST комментария возвращает код 201 (Created).
  3. В ответе комментария присутствуют post_id, author «Комментатор», content «Мой комментарий», id.
  4. GET списка комментариев возвращает один комментарий с текстом «Мой комментарий».


5. test_update_comment — проверка редактирования комментария.

действия:
  1. Создание поста через POST /api/posts.
  2. Создание комментария через POST /api/posts/{post_id}/comments (автор «Комментатор», текст «Комментарий»).
  3. Отправка PUT-запроса на /api/posts/{post_id}/comments/{comment_id} с новым текстом «Обновлённый комментарий».
  4. Проверка ответа.

ожидаемый результат:
  1. Создание поста и комментария возвращают 201.
  2. PUT возвращает код 200 (OK).
  3. В JSON-ответе PUT поле content равно «Обновлённый комментарий».


6. test_delete_comment — проверка удаления комментария.

действия:
  1. Создание поста через POST /api/posts.
  2. Создание комментария через POST /api/posts/{post_id}/comments (автор «Комментатор», текст «Комментарий»).
  3. Отправка DELETE-запроса на /api/posts/{post_id}/comments/{comment_id}.
  4. Отправка GET /api/posts/{post_id}/comments.

ожидаемый результат:
  1. Создание поста и комментария возвращают 201.
  2. DELETE возвращает код 204 (No Content).
  3. GET списка комментариев возвращает пустой список (длина 0).
