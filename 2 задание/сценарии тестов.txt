сценарий тестов

1. test_task_list_view_returns_200 - проверка, что главная страница открывается без ошибок
действия:
  1. переход на главную страницу
  2. отправка GET-запроса на URL списка задач.
ожидаемый результат:
  1. сервер возвращает код ответа 200 (OK).
  2. страница со списком задач отображается.


2. test_task_list_shows_tasks - проверка, что на странице списка отображаются созданные задачи.

действия:
  1. В базе данных создаётся задача с заголовком «Тестовая задача» и описанием «Описание».
  2. Пользователь открывает страницу списка задач (GET-запрос).
  3. Проверяется содержимое ответа.

ожидаемый результат:
  - В HTML-ответе присутствует заголовок задачи «Тестовая задача».
  - В HTML-ответе присутствует описание «Описание».


3. test_task_create_get_returns_200 - проверка, что страница создания новой задачи открывается без ошибки.

действия:
  1. Пользователь переходит на страницу «Новая задача» (форма создания).
  2. Браузер отправляет GET-запрос на URL создания задачи.

ожидаемый результат:
  - Сервер возвращает код ответа 200 (OK).
  - Отображается форма с полями «Заголовок» и «Описание».



4. test_task_create_post_creates_task - проверяет добавление задачи

действия:
  1. отправляет POST-запрос с данными: заголовок «Новая задача», описание «Текст», выполнено — нет.
  2. сервер обрабатывает форму и создаёт запись в базе.
  3. проверяется редирект и данные в базе.

ожидаемый результат:
  - код ответа 302 (редирект)
  - редирект ведёт на страницу списка задач
  - в базе ровно одна задача
  - у созданной задачи: заголовок «Новая задача», описание «Текст», флаг «выполнено» = False


5. test_task_detail_returns_200_and_shows_task - проверка, что страница просмотра одной задачи открывается и показывает её данные.

действия:
  1. В базе создаётся задача «Задача для просмотра» с описанием «Описание».
  2. Пользователь переходит на страницу этой задачи (GET по её id).
  3. Проверяется код ответа и содержимое страницы.

ожидаемый результат:
  - Код ответа 200 (OK).
  - На странице отображается заголовок «Задача для просмотра».
  - На странице отображается описание «Описание».


6. test_task_detail_404_for_invalid_pk - проверка, что при запросе несуществующей задачи возвращается ошибка 404.

действия:
  1. Пользователь переходит по URL задачи с несуществующим id (например, 99999).
  2. Браузер отправляет GET-запрос.
  3. Сервер ищет задачу в базе и не находит.

ожидаемый результат:
  - Сервер возвращает код ответа 404 (Not Found).
  - Пользователь видит страницу ошибки «Задача не найдена» (или стандартную 404).


7. test_task_update_get_returns_200 - проверка, что страница редактирования задачи открывается и в форме подставлены текущие данные.

действия:
  1. В базе создаётся задача с заголовком «Исходная» и описанием «Описание».
  2. Пользователь переходит на страницу редактирования этой задачи (GET).
  3. Проверяется код ответа и наличие заголовка задачи в форме.

ожидаемый результат:
  - Код ответа 200 (OK).
  - В ответе присутствует текст «Исходная» (форма заполнена данными задачи).


8. test_task_update_post_updates_task - проверка, что отправка формы редактирования обновляет задачу и перенаправляет на список.

действия:
  1. В базе создаётся задача «Было» с описанием «Старое».
  2. Пользователь отправляет POST-запрос на URL редактирования с новыми данными: заголовок «Стало», описание «Новое».
  3. Сервер обновляет запись в базе и выполняет редирект.
  4. Проверяются данные задачи после обновления.

ожидаемый результат:
  - Код ответа 302 (редирект).
  - Редирект ведёт на страницу списка задач.
  - В базе у задачи заголовок «Стало», описание «Новое».

9. test_task_delete_get_returns_200 - проверка, что страница подтверждения удаления открывается и показывает название задачи.

действия:
  1. В базе создаётся задача «На удаление» без описания.
  2. Пользователь переходит на страницу удаления этой задачи (GET).
  3. Проверяется код ответа и текст на странице.

ожидаемый результат:
  - Код ответа 200 (OK).
  - На странице отображается вопрос об удалении и название задачи «На удаление».


10. test_task_delete_post_deletes_task - проверка, что подтверждение удаления удаляет задачу из базы и перенаправляет на список.

действия:
  1. В базе создаётся задача «Удаляемая».
  2. Пользователь отправляет POST-запрос на URL удаления этой задачи (подтверждение).
  3. Сервер удаляет задачу и выполняет редирект.
  4. Проверяется количество задач в базе.

ожидаемый результат:
  - Код ответа 302 (редирект).
  - Редирект ведёт на страницу списка задач.
  - В базе не остаётся ни одной задачи (count = 0).
