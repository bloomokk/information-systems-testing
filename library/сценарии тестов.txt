Сценарии тестирования

Сценарий 1. Добавление книги через форму

Название теста: test_add_book

Цель тестирования: Убедиться, что при добавлении книги через форму данные сохраняются в базе данных и у созданной книги корректно заполнены поля: название, автор, год издания.

Предпосылки:
  Приложение запущено и доступно для тестирования.
  База данных доступна (миграции применены).
  Страница добавления книги доступна по URL.

Шаги тестирования:

  1. Отправить POST-запрос на URL добавления книги с данными: название «Война и мир», автор «Л. Толстой», год «1869».

  Ожидаемый результат: Сервер обрабатывает запрос и выполняет редирект после сохранения (код ответа 200 при следовании редиректу).

  2. Проверить количество записей книг в базе данных.

  Ожидаемый результат: В базе данных ровно одна книга.

  3. Получить созданную книгу из базы и проверить значения полей: название, автор, год.

  Ожидаемый результат: У книги заголовок «Война и мир», автор «Л. Толстой», год 1869.

Фактический результат: Заполните после выполнения теста.

  Ожидаемый результат: Сервер возвращает код ответа 200 (OK).

  Ожидаемый результат: В базе данных нет ни одной книги (количество равно 0).

Сценарий 5. Выдача книги читателю

Название теста: test_issue_book

Цель тестирования: Убедиться, что при оформлении выдачи книги читателю в базе создаётся запись выдачи с указанием книги и читателя, при этом дата возврата не заполнена (книга на руках).

Предпосылки:
  Приложение запущено и доступно для тестирования.
  В базе есть хотя бы одна книга и один читатель (для теста: книга «Книга», автор «Автор»; читатель «Иван Иванов»).
  Книга не выдана (нет активной выдачи по этой книге).

Шаги тестирования:

  1. Убедиться, что в базе есть книга и читатель. Отправить POST-запрос на URL выдачи книги, указав выбранную книгу и читателя.

  Ожидаемый результат: Сервер возвращает код ответа 200 (OK).

  2. Проверить количество записей выдачи (BookLoan) в базе данных.

  Ожидаемый результат: В базе ровно одна запись выдачи.

  3. Получить созданную запись выдачи и проверить: привязаны правильные книга и читатель; поле «дата возврата» пустое.

  Ожидаемый результат: У записи выдачи указаны корректные книга и читатель; дата возврата не заполнена (книга на руках).

Фактический результат: Заполните после выполнения теста.

  Ожидаемый результат: Поле «дата возврата» заполнено (не пустое).

Сценарий 7. Доступность книги для выдачи после возврата

Название теста: test_after_return_book_available_for_issue_again

Цель тестирования: Убедиться, что после возврата книги она снова появляется в списке книг, доступных для выдачи (при оформлении новой выдачи).

Предпосылки:
  Приложение запущено и доступно для тестирования.
  В базе есть книга, читатель и активная выдача этой книги (книга на руках).

Шаги тестирования:

  1. Создать выдачу книги читателю (книга на руках).

  Ожидаемый результат: Запись выдачи создана, книга числится выданной.

  2. Отправить POST-запрос на URL возврата этой выдачи.

  Ожидаемый результат: Возврат выполнен успешно (код 200).

  3. Открыть страницу формы выдачи книги (GET-запрос на URL формы выдачи).

  Ожидаемый результат: Страница открывается, код ответа 200.

  4. Проверить список книг в поле «книга» формы выдачи.

  Ожидаемый результат: Возвращённая книга присутствует в списке доступных для выдачи книг.

Сценарий 8. Выданная книга не доступна для повторной выдачи

Название теста: test_issued_book_not_in_issue_form_choices

Цель тестирования: Убедиться, что книга, которая уже выдана и не возвращена, не отображается в списке книг при оформлении новой выдачи (нельзя выдать одну и ту же книгу дважды одновременно).

Предпосылки:
  Приложение запущено и доступно для тестирования.
  В базе есть книга, читатель и активная выдача этой книги (книга выдана, не возвращена).

Шаги тестирования:

  1. Создать в базе запись выдачи: книга выдана читателю, дата возврата не заполнена.

  Ожидаемый результат: Книга числится выданной.

  2. Отправить GET-запрос на страницу формы выдачи книги.

  Ожидаемый результат: Страница формы выдачи открывается (код 200).

  3. Проверить список книг в поле «книга» формы выдачи.

  Ожидаемый результат: Выданная книга отсутствует в списке доступных для выдачи; её нельзя выбрать для новой выдачи.

Фактический результат: все тесты работают без ошибок
