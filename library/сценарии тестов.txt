Сценарий тестов

1. test_add_book — проверка, что при добавлении книги через форму книга сохраняется в БД с правильными полями.

действия:
  1. Отправка POST-запроса на URL добавления книги с данными: название «Война и мир», автор «Л. Толстой», год «1869».
  2. Проверка кода ответа и состояния базы данных.

ожидаемый результат:
  1. Сервер возвращает код ответа 200 (OK).
  2. В базе данных ровно одна книга.
  3. У созданной книги заголовок «Война и мир», автор «Л. Толстой», год 1869.


2. test_add_book_empty_year_ok — проверка, что книгу можно добавить без года издания (год может быть пустым).

действия:
  1. Отправка POST-запроса на URL добавления книги с данными: название «Книга без года», автор «Автор», год пустой.
  2. Проверка кода ответа и значения поля «год» у созданной книги.

ожидаемый результат:
  1. Сервер возвращает код ответа 200 (OK).
  2. В базе данных ровно одна книга.
  3. Поле «год» у созданной книги равно None (пусто).


3. test_delete_book — проверка, что при удалении книги по POST-запросу книга исчезает из базы данных.

действия:
  1. В базе данных создаётся книга с названием «Удаляемая», автором «Автор» и годом 2000.
  2. Отправка POST-запроса на URL удаления этой книги.
  3. Проверка количества книг в базе.

ожидаемый результат:
  1. Сервер возвращает код ответа 200 (OK).
  2. В базе данных нет ни одной книги (количество книг равно 0).


4. test_delete_book_get_shows_confirm — проверка, что страница удаления книги (GET) показывает подтверждение с названием книги.

действия:
  1. В базе данных создаётся книга с названием «Удаляемая» и автором «Автор».
  2. Отправка GET-запроса на URL удаления этой книги.
  3. Проверка кода ответа и содержимого HTML-ответа.

ожидаемый результат:
  1. Сервер возвращает код ответа 200 (OK).
  2. В HTML-ответе присутствует название книги «Удаляемая» (страница подтверждения удаления отображается).


5. test_issue_book — проверка, что при выдаче книги читателю создаётся запись выдачи и книга числится на руках.

действия:
  1. В базе создаются книга («Книга», автор «Автор») и читатель («Иван Иванов»).
  2. Отправка POST-запроса на URL выдачи книги с выбранной книгой и читателем.
  3. Проверка кода ответа, количества записей выдачи и полей созданной записи.

ожидаемый результат:
  1. Сервер возвращает код ответа 200 (OK).
  2. В базе данных ровно одна запись выдачи (BookLoan).
  3. У записи выдачи указаны правильные книга и читатель.
  4. Поле «дата возврата» у записи пустое (книга на руках).


6. test_return_book — проверка, что при возврате книги у записи выдачи проставляется дата возврата.

действия:
  1. В базе создаются книга, читатель и запись выдачи (книга выдана, дата возврата пустая).
  2. Отправка POST-запроса на URL возврата этой выдачи.
  3. Обновление объекта записи из БД и проверка поля «дата возврата».

ожидаемый результат:
  1. Сервер возвращает код ответа 200 (OK).
  2. У записи выдачи поле «дата возврата» заполнено (не None).


7. test_after_return_book_available_for_issue_again — проверка, что после возврата книга снова появляется в списке доступных для выдачи.

действия:
  1. В базе создаются книга, читатель и запись выдачи (книга выдана).
  2. Отправка POST-запроса на URL возврата книги.
  3. Отправка GET-запроса на страницу формы выдачи книги.
  4. Проверка списка книг в поле «книга» формы.

ожидаемый результат:
  1. Сервер возвращает код 200 при открытии формы выдачи.
  2. Возвращённая книга присутствует в списке доступных книг (в queryset поля «книга» формы).


8. test_issued_book_not_in_issue_form_choices — проверка, что выданная книга не показывается в выборе книги при оформлении новой выдачи.

действия:
  1. В базе создаются книга, читатель и запись выдачи (книга выдана и не возвращена).
  2. Отправка GET-запроса на страницу формы выдачи книги.
  3. Проверка списка книг в поле «книга» формы.

ожидаемый результат:
  - Выданная книга отсутствует в списке доступных для выдачи (не входит в queryset поля «книга» формы).
